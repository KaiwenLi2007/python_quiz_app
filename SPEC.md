Python CLI Quiz App

Startup: User runs python main.py.

Login: The app asks: "Wanna (1) Log in or (2) Register?".

(2)
Register: User makes a username and a password. Save it securely.

(1)
Log in: User enters info. Check if it matches what's saved.

Main Menu: Once logged in, say hi by name and show:
[1] Take a Quiz
[2] View Stats
[3] Exit

Quiz Setup: If they want to play, ask how many questions.

Smart Shuffle: Pick that many questions. If they previously said they liked a question, make it more likely to show up again.

Playing the Game: For each question:

Show the category, the question, and the type (Multiple Choice, True/False, or Short Answer).

Extra Feature: Ask: "Want a hint for 0.5 points? (y/n)". Show the hint if they type 'y'.

Ask them for an answer.

Grade it right away. 1 point for correct (0.5 if they used a hint), 0 for wrong. Tell them the right answer if they messed up.

Feedback: Ask: "Did you like this question? (y/n)". Save their answer to help pick future questions.

Wrapping Up: Tell them their final score (like "4.5/5.0!"), save their stats quietly in the background, and loop back to the Main Menu.

The Data
We need two main files to hold data.

A. The Questions (questions.json)
A normal JSON file anyone can read. We added a "hint" field to the base format.

{
"questions": [
{
"question": "What keyword is used to define a function in Python?",
"type": "multiple_choice",
"options": ["func", "define", "def", "function"],
"answer": "def",
"category": "Python Basics",
"hint": "It is a three-letter abbreviation of the word 'define'."
},
{
"question": "A list in Python is immutable.",
"type": "true_false",
"answer": "false",
"category": "Data Structures",
"hint": "Think about whether you can use the .append() method on a list."
},
{
"question": "What built-in function returns the number of items in a list?",
"type": "short_answer",
"answer": "len",
"category": "Python Basics",
"hint": "It is short for 'length'."
}
]
}


(Assume there are more questions in the real file, just use this structure!)

B. The User Data (users.dat)
This file should look like gibberish to a human (use pickle). Also, hash the passwords with hashlib (like SHA-256) so they are actually safe. It needs to track:

Hashed passwords.

Total quizzes, total points, and average score.

A record of which questions the user liked/disliked so the smart shuffle works.

File Setup
Keep the code clean by splitting it up:

main.py: The boss file. Handles the menus and gets user input.

auth.py: Handles logging in, registering, and hashing passwords.

quiz_engine.py: Handles picking questions, scoring, and hints.

storage.py: Deals with reading/writing to questions.json and users.dat.

questions.json: The human-readable questions file.

users.dat: The secret, messy-looking file generated by the code.

What if things break? (Error Handling)

No Questions File: If questions.json goes missing, catch the FileNotFoundError, print "Oops! Can't find questions.json" and exit(1).

Bad Typos: If they are supposed to type a number and they type "cat", or pick option "Z" on a multiple-choice question, catch it. Say "Invalid input" and let them try again. Don't punish their score for typos.

Corrupt Save File: If users.dat breaks or won't unpickle, don't crash. Print a warning, wipe it, and start fresh with a blank database.

Done Checklist (Acceptance Criteria)

App complains and exits nicely (code 1) if questions.json isn't there.

I can register, close the terminal, open it again, and log in.

Opening users.dat in Notepad shows unreadable garbage (no plain passwords!).

Typing letters instead of numbers for "how many questions" asks me to try again instead of crashing.

Saying "y" to a hint actually shows the hint and limits that question's score to 0.5.

Liking a question updates the hidden save file.

The code actually checks past likes/dislikes when picking new questions.
